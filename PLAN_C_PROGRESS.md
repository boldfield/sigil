The dedicated trap code (separate from `TRAP_NONEXHAUSTIVE_MATCH = 0x41` and `TRAP_HANDLE_DISCIPLINE_VIOLATION = 0x42`) makes a regressed `sigil_panic` (e.g., a future version that returns instead of exiting) surface as a recognisable abort signature instead of a generic IR violation. `assert` lowers as a brif: the success branch jumps to a merge block carrying `unit` (I8); the fail branch calls `sigil_panic(msg)` + traps without jumping into the merge, so the panic-leg's value flow is independent of the success path. Both arms wired into the entry-walker globals set so they pre-pass through `block_unsupported_handle`.

    Test surface: 9 e2e behavioural tests in `compiler/tests/e2e.rs` — direct-`panic` exit + stderr message, A=Int instantiation in a let-binding, A=Bool instantiation (load-bearing for the sub-pointer-width path through `cranelift_ty_of_ty`; A=Int / A=String would all lower to I64 and miss any width-narrowing regression in the placeholder lookup), A=String in match-wildcard arm with both zero-path and panic-path covered, `assert(true,_)` no-op, `assert(false,_)` panic-leg, and `assert` from a callee fn frame in both passing and failing variants. - Task STATE-CELL (Plan C addendum) — switch `std/state.sigil` from Plotkin lambda-encoding to a runtime-cell encoding, closing the State+Raise composition SIGSEGV. Plan: `designs/done/2026-05-08-sigil-state-runtime-cell.md`.
  - status: **done-pending-ci** ([state-runtime-cell branch / PR #117]). Pre-fix, `catch(run_state(... let _v = perform State.get(); raise(...) ...))` SIGSEGV'd in main's tuple destructure of the `run_state` result. Root cause: each State arm body returned `fn(s: S) -> (A, S) ![| e]` (a state-fn closure capturing `k`); the lifted lambda is Sync per `compute_user_fn_abi`, and Sigil's Sync ABI cannot carry a `DISCHARGED` tag through fn returns — foreign discharges from inside `k(s)` silently lost the tag, leaving `state-fn(initial)` to return the discharged value typed-as-tuple. **Fix.** Replace the encoding: arms now resume `k(...)` directly, backed by a `Ref[S]` runtime cell:

    ```sigil
    State.set(arg, k) => k(__set_then_arg(cell, arg))
    ```
